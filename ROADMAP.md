# AiUE Roadmap

## M0 Repo Productization

- Freeze monorepo product shape
- Add governance, contributing, security, support, and release assets
- Freeze stable CLI and schema surface
- Add public repo templates and CI stubs

## M1 Hard Gates

- Add schema, bundle, workspace, and tripline checks
- Formalize development lanes and routing rules
- Formalize `Alpha with Hard Gates`

## M2 Data Contract Cleanup

- Standardize report envelope fields
- Document compatibility and legacy handling
- Reduce legacy null-key generation in workflow outputs

## M3 Open Alpha

- Publish `AiUE` as a monorepo
- Keep `PMXPipeline` as the first official workflow pack
- Use Discussions and workflow-pack proposals as collaboration entrypoints

## M4 Platform Expansion

- Add a second workflow pack or lab
- Prove the platform is broader than PMX import

## M5 Reliability Growth

- Expand hosted regression coverage
- Improve cross-machine reproducibility
- Tighten release gates as stable surface matures
